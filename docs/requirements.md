# sj-fresh 요구사항

> 냉장고 재료 관리 웹앱. 모바일에서도 쓸 수 있게 반응형 웹.

## MVP (1차 목표 — 꼭 되어야 함)

### 재료 관리
- [x] 재료 추가: 이름, 양(자유 텍스트), 날짜, 메모 입력
  - 날짜는 **유통기한** 또는 **개봉일자** 중 선택 (유통기한 없는 재료용)
- [x] 재료 목록 조회 (유통기한 임박 순으로 정렬)
- [x] 재료 카드 탭 → 수정 팝업 (모든 필드 수정 가능, 메모 포함)
- [x] 재료 "소비" 처리 → soft delete (DB에서 삭제 X, `consumed_at` 타임스탬프 기록)
- [ ] 소비 이력 조회 (언제 뭘 썼는지)

### 플랫폼
- [ ] 모바일 웹 반응형 (폰에서 쾌적하게 쓸 수 있어야 함)
- [ ] PC 브라우저도 정상 동작

## 옵션 (여유 되면 추가)

### 음성 입력
- [ ] "우유 넣었어, 유통기한 1주일" 같이 말하면 재료 추가
- [ ] "우유 다 먹었어" 하면 소비 처리
- 기술: Web Speech API (브라우저 내장)

### 사진
- [ ] 재료 추가 시 사진 첨부
- [ ] 목록에서 썸네일로 표시
- 저장소: Supabase Storage

### 알림 (추후)
- [ ] 유통기한 임박 알림 (푸시 또는 이메일)

## 명시적 비범위 (안 할 것)
- 여러 사용자/가족 공유 — 1인 사용 기준으로 먼저
- 재료 바코드 스캔 — 음성이 우선순위 높음
- 레시피 추천 — 별개 기능, 나중에

## 설계 원칙
- **절대 hard delete 하지 않는다.** 모든 "삭제"는 `consumed_at` 또는 `deleted_at` 타임스탬프 기록으로 처리.
- 모바일 퍼스트로 UI 설계. PC는 그 다음.
- 로그인은 최대한 나중에. 처음엔 단일 사용자 가정.
